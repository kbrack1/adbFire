The "Backup Kodi" button backs up the contents of the .kodi folder on the device. Please note that Kodi MUST have been run for the first time before using this function. Empty directories are not backed up. If you are running something other than Kodi you will need to change the package name amd File Path in the Device Setup dialog to whatever package name and File Path your software uses.

If you have moved data or thumbnails to extUsb, the backup still proceeds normally. However in the case of thumbnails, they are saved in the default userdata folder of the backup and the advancedsettings.xml file is renamed advancedsettings,backup.

Currently, when restoring backup sets nothing is moved to extUsb. If desired you will need to redo the move via adbFire. A future release is planned that will perform data/thumbnail moves as part of the restore.

If you have questions or comments, please email jocala@jocala.com or visit one of the forums linked below. adbFire produces a log file, adbfire.log, with each run. Include it in your email or post if you're having problems.
