USB Mode
USB on the Stick is used when a normal connection cannot be made. Getting a USB connection is not foolproof/. You may have to check/uncheck the USB mode checkbox several times, stop/start the ADB server several times or unplug/plugin the Stick before a connection locks in.

In order to connect Windows to the Stick via USB, Google's WinUSB device driver must be installed. This driver is included with adbFire in the "winusb" folder. Directions for installation can be seen at jocala.com; the link is below this help doc. You can also see its installation in adbFire's Video Help.

The USB Mode checkbox allows adbFire to perform all non-root functions via a USB connection to the Fire TV Stick. It does not work with the standard Fire TV.

The connect/disconnect buttons are disabled when the USB Mode box is checked. A USB connection is obtained when the ADB server detects the Stick connected via USB.

You may also try the File->Console command. This opens a terminal session where you can issue adb commands via the command-line to establish a connection, but if you make a USB connection via console will have to check the USB via Console checkbox to tell adbFire that a USB connection has been manually obtained.

The USB via Console checkbox tells adbFire that a connection has been made manually, using the adbFire Console. The USB mode checkbox, and the connect/disconnect buttons are disabled when this checkbox is clicked.

In either case, when you request a USB connection a dialog will pop up asking you to select your media center software packagename and filepath. You can select from the list of presets, or select other and fill in the values manually.

If you have questions or comments, please email jocala@jocala.com or visit one of the forums linked below. adbFire produces a log file, adbfire.log, with each run. Include it in your email or post if you're having problems.
