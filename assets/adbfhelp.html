adbFire v1.10 is a companion program for Android XBMC and its forks. It also has many special features for the AFTV.

adbFire allows you to install (sideload) XBMC and other apps to your Android device without the bother of installing the Android SDK.

Additionally, it allows you to mount USB drives (when rooted) copy files to and from XBMC, backup and restore XBMC's data, root the AFTV (51.1.1.0 or earlier) and much more.

adbFire's non-device-specific features have been successfully tested on Samsung cell phones and tablets.

Prior to using adbFire there are setup steps required on the AFTV itself:

Set up AFTV
From the main (Launcher) screen of AFTV, select Settings.

Select System > Developer Options.

Select ADB Debugging.

Get the IP Address
 From the main screen of AFTV, select Settings.

Select System > About > Network. Make note of the IP address listed on this screen.

adbFire Startup
At startup "ADB running" is displayed the status bar. If it's not, click "Start" in the ADB server group.

Enter the Android device IP or name into the text box (e.g. 192.168.1.110, or mydevice, if you have assigned your device a name). This address is "sticky", it will be remembered for future runs.

Click "Connect". You should see "Device connected" on the status bar. If you don't, double-check your entry in the text box. When you establish a successful connection, adbFire will remember it for your next session.

If you're unable to start the ADB server and connect to your Android device, this program will be non-functional. Please contact the author, jocala@jocala.com so your problem can be fixed!

ADB Server
The "Start" button starts the adb server
The "Stop" button stops the adb server

Push/Pull/Delete

NOTE: Before pushing/pulling/deleting data in XBMC folders, XBMC MUST have been run for the first time on the Android device. XBMC sets up its inner directory structure when first run. The folder /sdcard/ is available anytime.

"Push" and "Pull" are simply Android's terminology for file copying. From your PC you "push" files to your Android device or conversely, you "pull" files from your Android device to your PC.

To push files select the destination folder from the dropdown widget, then press "Push". Next, choose and confirm the file to push from PC to device. The source folder is "sticky", it will be remembered for future runs.

To pull files select the source folder from the dropdown widget, then press "Pull". You will be presented with a browsable list of files. Select the file you wish to pull and double-click or press "Ok". It will be copied to the PC's "pull folder" (default: home folder). You can change the destination folder for pulled files in the preferences dialog.

To delete files select the source folder from the dropdown widget, then press "Delete". You will be presented with a browsable list of files. Select the file you wish to delete and double-click or press "Ok".

"USB Drive/Samba" This mounts/unmounts all connected USB drives or sticks. If checked, the "persistence" checkbox mounts your devices at boot.

Mounting USB mass storage devices requires installation of busybox.

Mounted drives (and /sdcard) are exported to the network via Samba, a form of the SMB/CIFS networking protocol. The Samba server you should see on your network is called adbFire. The storage is exported as sdcard and usb. Usb is a folder containing all your /storage/usb/sd* devices.

Other Buttons
The "Install APK" button installs or "sideloads" apps. Click it and choose the APK file you want installed. All installs are done with the "-r" flag.

This has the effect of preserving a previously installed app's data. It has no effect when installing a new app. If you require a "clean" install of an existing app, uninstall it first.

The apk folder is "sticky", it will be remembered for future runs.

The "Uninstall APK" button uninstalls non-system apps from your Android device. Double-click or highlight apps then click "ok" to select. To retain application data tick the "-k" checkbox. Data retention has only been tested with XBMC.

The "Backup XBMC" button backs up the org.xbmc.xbmc folder on the device. Please note that XBMC MUST have been run for the first time before using this function. Empty directories are not backed up.

The "Restore XBMC" button restores a backed up copy of org.xbmc.xbmc (or fork) to the device. XBMC must be closed in order to restore.

The "Push Remote" button installs an xml file defining a remote for XBMC.

The remote xml files are located in a folder named "xml" in the adbfiles folder. (For OS X, the adbfiles folder is inside the adbFire.app bundle. Right click on this file to browse its contents.) Three xml files for the AFTV remote are included with adbFire (source: XBMC Wiki). Any xml file installed with this function is installled to userdata/keymaps/keyboard.xml. Please note that XBMC MUST have been run for the first time before using this function.

The "Splash Screen" button installs a png splash screen file to the "media" folder under .xbmc. A Kodi splash screen and Elmerhuso's Gotham splash screen are included.

You may also drop your own png files into the adbfiles/splash folder (For OS X, the adbfiles folder is inside the adbFire.app bundle. Right click on this file to browse its contents). Please note that XBMC MUST have been run for the first time before using this function.

Filesystem R/W Makes /system partition writable.

Filesystem R/O Makes /system partition read-only.

The "Screen shot" button takes a snapshot of the device's current screen, gives it a date/time based name, and saves it to the pulled files folder (default: home directory).

Unfortunately, High-bandwidth Digital Content Protection (HDCP) prevents you from taking screenshots of most commercia video streams. Luckily, XBMC is not affected by HDCP.

Llama is an app that tells a device to perform tasks when certain conditions are met. It's very handy in controlling the AFTV. The "Llama Options" button opens a dialog that offers different Llama related events:

The "Install Llama" Click this checkbox to install Llama.

1. Boot the AFTV to XBMC
Starts XBMC when the AFTV boots.

2. Link XBMC to ikono TV
It's very irritating to have to drill down seven or eight steps to start sideloaded apps on the AFTV. If you install the app "ikono TV" from the Amazon Store, Llama can hijack it, so when you press the ikono icon, XBMC starts. This way you have XBMC on the "Recent" and "Apps" menu of the AFTV. You may see the ikono TV app flash briefly before XBMC starts.

3. Both
XBMC starts when the AFTV boots AND is linked to the ikono TV app.

4. No action
Do not adjust Llama settings.

The next four options relate to the ikono TV icon.

1. Replace ikono TV icon with XBMC icon
2. Replace ikono TV icon with Kodi icon
3. Restore default ikono TV icon
4. No action (the default)

Note: After installing Llama and selecting an event, you must run Llama and select "import event from USB". The events that adbFire installs start XBMC. They will need to be edited to start variants like SPMC.

The "Root AFTV" button installs Chainfire Superuser and GeoHot's towelroot v 3.0. You must complete the process by running the towelroot app. Unfortunately, firmware 51.1.3.0 and above can't be rooted at this time. Check your Fire TV Settings/About to see your firmware ID.

The "ADB Shell" button opens a console (Terminal.app for OS X or cmd.exe for Windows) and runs adb shell against the connected device.

NOTE: Currently, under OS X, shell will fail if more than one device is detected by adb. e.g. adb devices reveals a phone AND a AFTV.

Preferences

XBMC Package name, default: org.xbmc.xbmc. This varies depending on what variant of XBMC you're running.

If you are running a fork of XBMC, enter its packagename here, e.g. for Beyond XBMC enter "com.flumpster.xbmc".

Thus far, SPMC, BeyondXBMC, elmerohueso's XBMC Launcher, Gotham 13.1 and Helix nightlies have been tested with adbFire.

In theory any fork should work if its file directory structure adheres to XBMC's (/sdcard/Android/data/packagename/files/.xbmc) directory structure. YMMV.

Pull Folder, default: User's home directory.
Choose the folder where pulled XBMC files and screenshot pngs are placed.

AFTV updates Toggle AFTV updates on/off. This disables/enables com.amazon.dcp. Default is updates on. Changing this option requires root.

SSH server at boot When checked this option has the SSH server start at boot. This option requires root.

The Check for updates at startup checkbox causes adbFire to check jocala.com for updates at startup. This is on by default.

The Check for updates button contacts jocala.com for an immediate update check. If an update is available adbFire offers to connect for download.

Root Menu items

These items only work with root.

Install Busybox
BusyBox combines tiny versions of many common UNIX utilities into a single small executable. It provides replacements for most of the utilities you usually find on a Linux system. It has an extremely small footprint, under 2Mb

Uninstall Busybox
Removes BusyBox from your system.

Install SSH
This option installs the Dropbear SSH server on your device. This allows you to open a secure shell on your Fire TV, somewhat like adb shell. Among many other things, it allows you to copy files to your device as root using the scp command and act as a backup entryway into your system in the case of a problem with adb.

SSH can be made persistant by checking the "SSH at boot" option in adbFire's preferences dialog. If you choose not to start SSH at boot it can be started and stopped via the command line:

Press "ADB Shell" then enter the command "sshstart password" where 'password' is the password you want SSH to use.

Stopping SSH is done via the command "sshstop", and you can find SSH's status with the command: "sshstatus".

Windows users will want to download the puTTY client, OS X and Linux users have default ssh clients.

Uninstall Busybox
Removes SSH from your system.

Install AFTV firmware
Install firmware to AFTV. Upgrade (or downgrade) your AFTV firmware. This works for firmware on a mounted USB drive/stick and for firmware on your PC.

When updating firmware from USB storage, adbFire looks on all devices for files with an extension of ".bin" or ".zip" and presents them for your choice.

Installing firmware from a usb stick/drive is nearly twice as fast as installing a firmware file located on your PC or LAN.

Please note that installation of firmware via Amazon's Native Recovery (this method) has been effectively obsoleted with the availibility of ClockworkMod, compiled for the Fire TV by rbox@xda.

IMPORTANT: Do not attempt to install custom Clockwork Mod firmware via Amazon Native Recovery. It will not work!

Install Recovery
Upgrade your AFTV recovery partition to ClockworkMod or return it to stock recovery.

The recovery partition on your Fire TV is a subsystem where system upgrades and other operations are performed. In effect, it's a "mini" operating system on your Fire TV.

As shipped by Amazon, the recovery partition is fairly useless; it allows the Fire TV to upgrade with offical Amazon images only. Thanks to rbox@xda we have a version of Koush's ClockworkMod Recovery, specially compiled for the Fire TV!

Clockwork Mod allows us to install custom pre-rooted images, backup our Fire TVs and more. adbFire includes (with permission from rbox) two recovery images:

firetv-cwm_recovery-6.0.5.1.2.img

recovery-stock-51.1.0.img

After clicking "Install Recovery" from the menu, select the image you wish to install, fire-cwm for Clockwork Mod or recovery-stock to restore Amazon's stock image.

To obtain updated recovery images click the CWM link below or the rbox link on the "Credits" adbFire menu item. You will be taken to rbox's CWM topic on xda where you will find image links.

Drop updated images into your "recovery" folder. For OS X users, right-click on adbFire.app and select "Show Package Contents" to browse to your recovery folder.

IMPORTANT: Do not attempt to install stock Amazon firmware via Clockwork Mod. It will not work! =============================

For Windows XP and up, the adbFire folder can be located wherever you wish.

Under OS X, adbFire.app may also be located wherever you wish.

adbFire keeps two log files adbfire.log and adbfire.log.old. Log files for Windows are kept in the adbFire folder. For OS X, right-click on adbFire.app and select "Show Package Contents". Browse to "Contents/MacOS/adbfiles".

The adb binaries and dlls are part of the Android SDK.
